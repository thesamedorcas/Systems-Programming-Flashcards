# Systems Programming Flashcards

Interactive flashcard deck covering all examinable topics for the Systems Programming course at the University of Glasgow.

## Usage

Open `index.html` directly in a browser — no server or dependencies needed.

**Navigation:**
- Click a card or press `Space` / `Enter` to flip
- Arrow keys `←` `→` or swipe to move between cards
- Use the tabs at the top to filter by topic

## Contents — 71 cards across 11 units

| Unit | Cards | Topics |
|------|-------|--------|
| Introduction | 7 | What is systems programming, history, paradigms, C vs Python, energy benchmarks, compilation pipeline, Makefiles |
| Basics of C | 14 | Data types, two's complement, variable lifetime, scope, arrays, structs, strings, functions, sizeof, static, common bugs |
| Memory & Pointers | 7 | Memory organisation, pointers, NULL, const, call-by-value vs reference, pointer arithmetic, common mistakes |
| Dynamic Memory | 4 | Stack vs heap, malloc/free, returning pointer to local variable, memory leaks |
| Data Structures | 6 | Linked list (full insert/delete/free), BST (search/insert), dynamic arrays, void* generics |
| Debugging | 6 | Syntactic vs semantic errors, segfaults, undefined behaviour, signed/unsigned overflow, string safety, memory tracing |
| Ownership & RAII | 3 | RAII pattern, unique_ptr/shared_ptr, Rule of Three/Five |
| Energy Efficiency | 2 | Power/energy/carbon metrics, cache-friendly code |
| Concurrency | 12 | Concurrency vs parallelism, processes vs threads, pthreads, race conditions, mutexes, deadlock, condition variables, complete programs |
| Synchronisation | 6 | Bounded buffer, monitors, semaphores, barriers, atomics, readers-writers |
| C++ Threading | 4 | std::thread, lock_guard vs unique_lock, condition_variable, async/future/promise |
